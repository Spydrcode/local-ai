## Technical Implementation Plan

### Priority 1: White-Label Branding System

**Files to Modify:**
1. [pages/api/export/[demoId].ts](pages/api/export/[demoId].ts)
   - Add `agencyBranding` parameter to PDF/PowerPoint generators
   - Replace hardcoded "We Build Apps" with dynamic agency name/logo

2. Create `lib/branding/agency-branding.ts`:
   ```typescript
   interface AgencyBranding {
     agencyName: string;
     logoUrl?: string;
     primaryColor: string;
     footerText: string;
   }
   ```

3. Update Supabase schema:
   ```sql
   CREATE TABLE agencies (
     id UUID PRIMARY KEY,
     name TEXT NOT NULL,
     logo_url TEXT,
     primary_color TEXT DEFAULT '#10b981',
     footer_text TEXT,
     created_at TIMESTAMPTZ DEFAULT NOW()
   );

   ALTER TABLE demos ADD COLUMN agency_id UUID REFERENCES agencies(id);
   ```

### Priority 2: Multi-Client Dashboard

**New Files to Create:**
1. `app/agency/dashboard/page.tsx` - Agency client list view
2. `app/agency/clients/[clientId]/page.tsx` - Individual client analysis hub
3. `components/ClientList.tsx` - Sortable/filterable client table

**Features:**
- Search clients by name/URL
- Filter by analysis status (complete, pending, error)
- Bulk actions (export 10 reports at once)
- Client tags (industry, priority, status)

### Priority 3: Team Collaboration

**Database Changes:**
```sql
CREATE TABLE team_members (
  id UUID PRIMARY KEY,
  agency_id UUID REFERENCES agencies(id),
  user_email TEXT NOT NULL,
  role TEXT CHECK (role IN ('owner', 'admin', 'member')),
  invited_at TIMESTAMPTZ DEFAULT NOW(),
  accepted_at TIMESTAMPTZ
);

CREATE TABLE activity_log (
  id UUID PRIMARY KEY,
  demo_id VARCHAR(15) REFERENCES demos(id),
  user_email TEXT NOT NULL,
  action TEXT NOT NULL, -- 'created', 'analyzed', 'exported'
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

**Features:**
- Email invitations to team members
- Role-based permissions (owner can invite, members can only analyze)
- Activity feed (who ran which analysis when)
